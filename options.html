Session Vault Settings

Auto-lock

The vault re-locks automatically to limit how long decrypted data stays in memory.

Change master password

Re-encrypts every saved session with the new password. This does not change any website logins — Session Vault only stores tab/window snapshots, not credentials.

Backup

Backups remain encrypted with your current master password — this file is safe to store, but you'll still need that password to restore it.

Danger zone

Permanently deletes the vault and every saved session from this browser. This cannot be undone unless you have an exported backup.

Session Vault stores everything locally, encrypted with AES-256-GCM. Nothing is ever sent over the network.
